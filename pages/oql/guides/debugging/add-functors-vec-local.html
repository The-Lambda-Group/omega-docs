add-functors: 03-vec on Local Instances

Visibility

Owner: omega | Workspace: OmegaAI | Visibility: Public

oql/queries/add-functors/03-vec.oql registers functor mappings for the vec (vector) datastore family. On local development instances that do not have a vec datastore configured, this file throws a WRITING_SYMBOLS error and does not complete. This doc records the observed behavior and the safe-to-skip conclusion for non-vec work.

What happens

Running omega-cli run-query oql/queries/add-functors/03-vec.oql on a local instance without a configured vec datastore produces an error of the form:

WRITING_SYMBOLS ["CompVecKey", "_"]

The file invokes Qo.Vec.Comp/add-functors (among others). When the vec datastore is absent, the symbol VectorToCompFunc remains unbound inside that clause, and the engine surfaces a WRITING_SYMBOLS error rather than completing normally.

The other eight add-functors files (00-tbl.oql through 02-dl.oql and 04-page-bl.oql through 08-run.oql) do not reference the vec datastores and succeed on the same local instance.

Root cause

Observed behavior only — root cause not fully confirmed. No minimal reproduction case has been established. The most likely explanation is that Qo.Vec.Comp/add-functors requires at least one vec datastore document to be populated before it can resolve VectorToCompFunc, and on a fresh local instance that document does not exist. The engine then fails when it tries to write functor symbols against an unresolved binding.

This is not a confirmed engine bug. It may be expected behavior for a feature that requires optional infrastructure (the vec datastore) which is not present in all local setups.

Is it safe to skip?

Yes, for any OQL work that does not involve vec functionality. Confirmed by grep: data-source/view.oql — the primary data-source query — has zero dependencies on any Qo.Vec.* namespace. Work on queries, data sources, tables, pages, implementations, or workflows that do not touch vec is unaffected by 03-vec's failure.

If your task is non-vec OQL work, skip 03-vec and run the remaining eight files. The add-functors sequence is not strictly ordered; 03-vec can be omitted without affecting the others.

When does 03-vec need to succeed?

03-vec must succeed when:

  • You are developing or debugging vec-related functionality (Qo.Vec.* namespaces).
  • You are verifying a full add-functors pass against a production-like local instance that has a vec datastore configured.
  • CI or a deployment pipeline requires all add-functors files to pass.

In these cases, you need a local instance with the vec datastore populated. Unknown: the exact setup steps to populate the vec datastore locally are not documented here — consult the vec-related OQL source or the platform team.
