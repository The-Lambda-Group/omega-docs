map-equality validation caveat

When you need to validate that two OQL result-key sets are the same, do not rely on direct map equality as the proof step. During MAB Critic validation work (2026-05-04), a probe comparing two small maps gave a misleading "equal" result at the point where a stricter key-set check was needed. The shipped fix avoided direct map equality and instead validated against sorted unique key lists. The underlying equality semantics for maps are Unknown — treat this as an empirical gotcha: direct map equality is unsafe for validator design.
