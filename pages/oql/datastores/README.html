Datastores & Storage

Data in OQL lives in datastores — named locations that hold terms (facts and rules). This section covers the full storage model: how to declare datastores and functors, how defterm primary-key semantics work and when adding one is destructive, and the singleton-row edge case where the index model has no clean answer.
