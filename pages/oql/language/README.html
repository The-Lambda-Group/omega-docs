OQL Language Reference

Core language constructs for writing OQL. These docs cover what the language provides and how it behaves — the authoritative reference for clause definition, control flow primitives, built-in terms, solution reducers, query execution strategies, and the hard rules that exist because every one of them has produced real data loss or silent wrong-answer bugs in production.

Visibility

Owner: omega | Workspace: OmegaAI | Visibility: Public

Contents

  • Clauses — Stored clauses, in-memory clauses, the clause term syntax, inner clauses, closures and capture semantics (including non-transitivity, per-clause scope, fold in clause bodies), and the five hard rules that must never be violated.
  • Control Flow — when/when-not/if universal quantifiers, with-table-if per-solution branching, schema binding rules, row-identity preservation, the drop idiom, and the not-operator absence.
  • Built-in Terms — Core terms, arithmetic, map operations, array operations, string operations, HTTP, symbols-in-literal-arguments rule, and deprecated http-get/post/delete terms.
  • Reducers — fold, with-unique-by, with-order-by, with-group-by, with-count, with-limit, with-skip, with-sort, and all fold pitfall patterns.
  • Query Execution — Index selection strategies (defterm PK, write-index exact/prefix, Mango fallback), operator push-down per strategy, the Mango silent-ignore gotcha for with-skip/with-limit, and the mango-query built-in with its mandatory enable-full-scan/disable-full-scan bookend requirement.
  • S-Expression Evaluation Model — Why S-expressions extend solution bindings rather than produce inline values, why map literals require pre-bound symbols (not nested term calls), and why terms like uuid must be wrapped in run-term for per-row execution.
